What is refactoring?

As described by Martin Fowler It is change made to the internal structure of a software program to make it easier to understand and modify without changing its overall behavior. As described by me refactoring is cleaning up the code

Why to Refactor?
  • It improves design of software
  • It helps prgoram eaiser to understand and modify
  • Sometimes helps find corner bug
When to Refactor
  • While fixing a bug
  • While adding feature to existing program
When not to refactor?
  • When we dont have a full understanding of what program does
  • When you dont have strong TEST CASE to confirm changes
How to decide whether a given program needs refactoring?
  • Duplicate Code
  • Long Method
  • Long Class
  • Feature Envy
  • Data Clump
